# Ho-06.3-AT-03 — Orphan remediation on unlinked-marker rows

**Goal**

The 06.2 "Unlinked markers" section stops being surfacing-only. Each
orphaned-marker row offers **Create Scope from Marker** — the AT-02 ingest
sheet seeded with the marker's directory and recorded scope ID — and
**Remove Stray Marker** — deletes the `.sharibako` file behind a
system-rendered destructive confirmation, rust-toned in-window per the 06.7
pattern. Scan-failure rows stay surfacing-only.

**Context**

ho-06.3 Decision 7 (`ho-process/hos/ho-06.3-first-run-and-ingest.md`). The
rows exist: `WorkshopModel+Glyphs.swift` derives `UnlinkedMarker.orphaned(scope:markerURL:)`
from the 06.1 scan cache and `ScopeSidebar.swift` renders the section
(surfacing-only, ho-06.2 Decision 2b). The destructive pattern is settled
(ho-06.7): in-window affordance wears `Color.drift`, the confirmation dialog
is system-rendered and untintable (06.5 gate — do not attempt to tint it),
pending-deletion state on the model, announce on the status surface.

The blast radius is INVERTED from 06.7's deletions: removing a marker
touches a file in the **user's repository**, not the vault. The confirmation
copy must say exactly that (the file's full path, "your project directory",
"the vault is untouched").

**Files**

- Modify: `Sources/Sharibako/Views/ScopeSidebar.swift` — the two affordances
  on `.orphaned` rows, following the section's existing row idiom (visible,
  titled — nothing buried in nested menus, the 06.2/06.5 lesson).
- Modify: `Sources/Sharibako/WorkshopModel+Glyphs.swift` or a new
  `WorkshopModel+Orphans.swift` if the file nears the length ceiling — the
  two tested intents + pending-removal state.
- Add tests: `Tests/SharibakoTests/` — both intents over temp directories.

**Required Changes**

1. **`createScopeFromMarker(_ marker:)`** — starts AT-02's ingest session on
   the marker's directory with the marker's recorded scope ID as the seeded
   suggestion (the marker names the scope the vault forgot; recreating under
   that ID is what heals the link). Rides `beginIngest`'s existing
   no-importable-secrets and reconcile behavior unchanged.
2. **`requestRemoveStrayMarker(_ marker:)` / `confirmRemoveStrayMarker()` /
   `dismissStrayMarkerRemoval()`** — the 06.7 pending-state trio. Confirm
   deletes the single `.sharibako` file (`FileManager`, no git, no vault
   write), refreshes the scan cache so the row leaves the section, announces
   with the path. Failure (file already gone, permissions) announces and
   leaves the model consistent.
3. **Row affordances** — Create Scope from Marker (accent voice) and Remove
   Stray Marker (`Color.drift`, rust) on `.orphaned` rows; `.confirmationDialog`
   system-rendered, `role: .destructive`, cancel default, copy naming the
   file path and the inverted blast radius. Scan-failure rows unchanged.
