### 2. `MaterializerTypes.swift` — public value types

All types `public`, `Sendable`, `Equatable`. Provide memberwise `public init` where the auto-synthesized version isn't accessible from outside the module.

```swift
public struct ScopeMarker: Sendable, Equatable, Codable {
    public let scope: String
    public let materializeTo: String?    // relative to marker directory; nil → "./.env"
    public let markerURL: URL            // not encoded; set by loader

    public var targetURL: URL {
        // Resolve materializeTo (or default) as file URL relative to marker's parent directory,
        // then standardize (resolve symlinks, etc.).
    }

    private enum CodingKeys: String, CodingKey {
        case scope
        case materializeTo = "materialize_to"
    }
}

public enum ScopeState: Sendable, Equatable {
    case liveHere(markerURL: URL, targetURL: URL)
    case liveElsewhere                       // scope exists in vault; no marker in configured roots
    case orphaned(markerURL: URL, reason: String)  // marker exists but vault has no such scope
}

public enum MaterializeResult: Sendable, Equatable {
    case wrote(path: URL, keysWritten: [String])
    case unchanged(path: URL)
    case diffPending(diff: MaterializeDiff)
}

public struct MaterializeDiff: Sendable, Equatable {
    public let scopeID: String
    public let path: URL
    public let ownedKeysDiffering: [String]         // sorted alphabetically
    public let ownedKeysMissingFromFile: [String]   // sorted alphabetically

    public init(scopeID: String, path: URL, ownedKeysDiffering: [String], ownedKeysMissingFromFile: [String])
}

public struct DriftReport: Sendable, Equatable {
    public let scopeID: String
    public let path: URL
    public let owned: [KeyDrift]
    public let parseWarnings: [ParseWarning]

    public init(scopeID: String, path: URL, owned: [KeyDrift], parseWarnings: [ParseWarning])
}

public enum KeyDrift: Sendable, Equatable {
    case match(key: String)
    case fileMissing(key: String)
    case fileValueDiffers(key: String, vaultSha256: String, fileSha256: String)
}

public enum CleanResult: Sendable, Equatable {
    case cleaned(path: URL, keysRemoved: [String], fileStillExists: Bool)
    case fileMissing(path: URL)
}

public struct ParseWarning: Sendable, Equatable {
    public let file: URL
    public let lineNumber: Int    // 1-indexed
    public let text: String       // the raw offending line
    public let reason: String

    public init(file: URL, lineNumber: Int, text: String, reason: String)
}
```

`ScopeMarker.targetURL` resolves `materializeTo` (or `"./.env"` if nil) via `URL(fileURLWithPath:relativeTo:)` against the marker's parent directory, then calls `.standardizedFileURL`.

### 3. `EnvParser.swift` — internal parser

Internal to `SharibakoCore` (no `public` on the types). AT-02 will import from the same target.

```swift
internal enum EnvLine: Sendable, Equatable {
    case blank(text: String)                                     // empty or whitespace-only
    case comment(text: String)                                   // leading whitespace + `#` prefix; whole line preserved
    case keyValue(key: String, value: String, rawText: String)   // rawText preserves the exact original bytes
    case malformed(text: String, reason: String)                 // preserved as-is; surfaced as ParseWarning
}

internal struct ParseResult {
    let lines: [EnvLine]
    let warnings: [ParseWarning]
}

internal func parseEnvFile(at url: URL) throws -> ParseResult
internal func parseEnvString(_ text: String, sourceFile: URL) -> ParseResult
```

Behavior specifics:

- **Line separator:** split on `\n`; strip a trailing `\r` if present (Windows CRLF support). Preserve the split behavior — do not preserve `\r\n` in `rawText`. The Materializer writes `\n` line endings on output.
- **BOM:** if the file starts with UTF-8 BOM (`0xEF 0xBB 0xBF`), strip it. Log a warning: "BOM stripped."
- **`export` prefix:** `export FOO=bar` and `export  FOO=bar` (multiple spaces) → parse as `keyValue(key: "FOO", value: "bar", rawText: "export FOO=bar")`. The `export` is preserved in `rawText` for non-owned lines but the Materializer's canonical rewrite drops it for owned lines (Decision 5 in the ho document).
- **Quoted values:**
  - `KEY=value` — bare
  - `KEY="value"` — double-quoted; supports `\\`, `\"`, `\n`, `\t` escapes
  - `KEY='value'` — single-quoted; no escapes, literal contents
- **Malformed rejections (all surface as ParseWarning):**
  - Lines starting with a digit for the key
  - Keys containing characters outside `[A-Za-z_][A-Za-z0-9_]*`
  - Values that open a quote but never close it
  - Values with backslash line continuations (unterminated line)
  - `KEY=` with a value that references `$VAR` or `$(cmd)` — flagged as unsupported interpolation
- **Comments after values:** `KEY=value # comment` — v1 does NOT support inline comments; the entire `value # comment` is treated as the value. This preserves the user's intent (in case they wanted the `#` literally). Consider revisiting post-v1.

`parseEnvFile(at:)` throws `VaultError.envParseFailed` only if the file cannot be read at all (encoding error, IO error). Malformed lines never throw — they return as `ParseWarning`.

### 4. `Materializer.swift` — main type + write-path operations

Public `struct Materializer`. Mirrors `Conduit`'s shape.

**Initializer:**

`public init(vaultCore: VaultCore, vaultURL: URL)` — stores both. Does not verify anything at init time (parallel to `VaultCore` and `Conduit` precedent).

**Marker helpers:**

```swift
/// Load a marker from a specific path.
public func loadMarker(at path: URL) throws -> ScopeMarker
```

Reads the file, decodes YAML via Yams + Codable to a temporary struct, then constructs `ScopeMarker` with the actual `markerURL`. Throws `markerNotFound` if the file doesn't exist. Throws `markerMalformed` if YAML fails to parse or `scope` field is missing/empty.

```swift
/// Write a marker at a specific path.
public func writeMarker(_ marker: ScopeMarker, at path: URL) throws
```

Encodes YAML via Yams. Atomic write (temp + rename).

```swift
/// Walk up from `startingFrom` looking for a `.sharibako` file.
/// Stops at home directory or filesystem root, whichever comes first.
public func resolveMarker(startingFrom: URL) throws -> ScopeMarker
```

Throws `markerNotFound(startingFrom:)` if the walk terminates without finding one.

**Scan:**

```swift
public func scan(roots: [URL]) throws -> [ScopeMarker]
```

For each root: `FileManager.enumerator(at:includingPropertiesForKeys:options:)` with `.skipsHiddenFiles = false` (we're looking for `.sharibako` which is hidden). Filter enumerator entries where the URL's `lastPathComponent == ".sharibako"`. Load each as a marker. Order results: breadth-first from each root, alphabetical within a depth. Deduplicate by marker's absolute path (in case scan roots overlap).

**Status:**

```swift
public func status(scopeID: String) throws -> ScopeState
```

Behavior:

1. Check if scope exists in vault (`vaultCore.listScopes()` or equivalent — use whatever existing operation returns scope IDs).
2. Scan configured roots for a marker with matching `scope:` field.
3. Cases:
   - Scope exists + marker found: `.liveHere(markerURL:, targetURL:)`
   - Scope exists + no marker found: `.liveElsewhere`
   - Scope missing + marker found: `.orphaned(markerURL:, reason: "vault has no scope named '<id>'")`
   - Scope missing + no marker: this shouldn't be called for a nonexistent scope, but throw `scopeNotFound` if it is.

For AT-01, take scan roots as a parameter: `public func status(scopeID: String, scanRoots: [URL]) throws -> ScopeState`. Scan-root config comes from app-level config which is ho-04 territory; the Materializer just takes the list.

**Materialize:**

```swift
public func materialize(scopeID: String, overwriteDrift: Bool = false) throws -> MaterializeResult
```

Algorithm (per Decision 5 in the ho document):

1. Resolve marker for the scope (call `status` with a callee-provided scanRoots — see below on callable-signature).
   - Actually: for AT-01, add a shorter method `resolveMarker(forScope: String, scanRoots: [URL]) throws -> ScopeMarker` and have `materialize` require the marker URL directly. `public func materialize(marker: ScopeMarker, overwriteDrift: Bool = false) throws -> MaterializeResult`. Reason: keeps the Materializer's dependency on scan-roots explicit at the call site, doesn't hide it.
2. Load the target file if present; parse to `ParseResult`.
3. Load owned keys and their vault values from `VaultCore`.
4. Compute drift:
   - `ownedKeysDiffering`: owned keys whose parsed file value differs from vault value.
   - `ownedKeysMissingFromFile`: owned keys with no `keyValue` line in the file.
5. If `ownedKeysDiffering` non-empty AND `overwriteDrift == false` → return `.diffPending(MaterializeDiff(...))`.
6. Otherwise, build output `[EnvLine]`:
   - Iterate the existing lines. For `keyValue(key:, value:, rawText:)` where key is owned: replace with a canonical `KEY=value` line (see canonical rewrite below).
   - All other lines pass through.
7. For owned keys still missing after step 6: append at the end. If the last line isn't `.blank`, add a `.blank` before them.
8. Render `[EnvLine]` to a string (each line's stored text + `\n`).
9. If the rendered text equals the original file's content byte-for-byte → return `.unchanged(path: marker.targetURL)`.
10. Otherwise, write atomically to the target path. Return `.wrote(path:, keysWritten:)` listing all keys whose lines were written or appended.

**Canonical rewrite:**

```swift
internal func canonicalizeEnvLine(key: String, value: String) -> String {
    // - If value is empty: KEY=
    // - If value contains whitespace, quotes, backslash, `#`, `$`, or newline: double-quote and escape
    // - Otherwise: KEY=value bare
}
```

Escapes inside double quotes: `\\`, `\"`, `\n`, `\t`. Nothing else. No single-quote canonical output (users' single-quoted lines round-trip through as-is because they're preserved in `rawText` for non-owned lines; owned lines get canonical double-quote or bare form on rewrite).

**Clean:**

```swift
public func clean(marker: ScopeMarker) throws -> CleanResult
```

Algorithm (per Decision 10):

1. If `marker.targetURL` doesn't exist → `.fileMissing(path: marker.targetURL)`.
2. Parse file.
3. Load scope's owned keys from VaultCore.
4. Filter out `keyValue(key:, ...)` where key is owned; preserve everything else.
5. If result is empty or contains only `.blank` / `.comment` → delete the file, return `.cleaned(path:, keysRemoved:, fileStillExists: false)`.
6. Otherwise render remaining lines to text, write atomically, return `.cleaned(path:, keysRemoved:, fileStillExists: true)`.

**Heal:**

```swift
public func heal(marker: ScopeMarker) throws -> DriftReport
```

Algorithm (per Decision 9):

1. If `marker.targetURL` doesn't exist → return DriftReport with every owned key as `.fileMissing`.
2. Parse file. Collect `ParseWarning`s.
3. Load owned keys + vault values.
4. For each owned key:
   - If not present in parsed lines → `.fileMissing(key:)`
   - If present + values match → `.match(key:)`
   - If present + values differ → `.fileValueDiffers(key:, vaultSha256:, fileSha256:)` — SHA-256 UTF-8 bytes of each value.
5. Return DriftReport.

Do NOT surface anything about non-owned keys or non-owned lines in DriftReport. Non-owned means invisible.

### 7. `Tests/…/EnvParserTests.swift` — parser suite

swift-testing `@Suite("Env Parser")`. Cover:

- Empty file → 0 lines
- Only blank lines → all `.blank`
- Only comments → all `.comment`
- Bare `KEY=value` → `.keyValue(key: "KEY", value: "value", rawText: "KEY=value")`
- `KEY=value with spaces` (no quotes) → parsed as `value with spaces`
- `KEY="value"` → parsed as `value` (quotes stripped)
- `KEY="value with \"quotes\""` → parsed as `value with "quotes"`
- `KEY='literal $value'` → parsed as `literal $value` (no interpolation)
- `export FOO=bar` → `.keyValue(key: "FOO", value: "bar", rawText: "export FOO=bar")`
- `KEY=  value  ` — leading/trailing whitespace in value: v1 preserves within the value (no trim). Document this in a test.
- BOM at start → stripped, warning emitted
- CRLF endings → parsed cleanly, warnings suppressed unless BOM
- Bad key: `1KEY=value` → `.malformed` + warning
- Unterminated quote: `KEY="oops` → `.malformed` + warning
- Multi-line value: `KEY="line1\nline2` continuing on next line → `.malformed` + warning
- Interpolation: `KEY=$OTHER` → `.malformed` + warning (unsupported)
- `KEY=` (empty value) → `.keyValue(key: "KEY", value: "", rawText: "KEY=")`
- Comment after value: `KEY=value # comment` → value is `value # comment` (whole thing after `=`)
- Repeated key: `KEY=first` then `KEY=second` — both parse as `.keyValue`; Materializer's merge uses the first when reading, replaces the first when writing. Document in tests.

### 8. `Tests/…/MaterializerWriteTests.swift` — write-path suite

swift-testing `@Suite("Materializer Write")`. Cover:

**Marker helpers:**
- `loadMarker` on missing file → throws `markerNotFound`
- `loadMarker` on valid YAML → returns marker with correct `targetURL`
- `loadMarker` on invalid YAML → throws `markerMalformed`
- `loadMarker` on YAML missing `scope` → throws `markerMalformed`
- `writeMarker` round-trips (write then load)
- `resolveMarker` walks up to find a marker
- `resolveMarker` throws `markerNotFound` when nothing is found up to root

**Scan:**
- `scan(roots:)` finds a single marker in the root
- `scan(roots:)` finds nested markers (marker at `root/project/.sharibako`)
- `scan(roots:)` returns empty on a root with no markers
- `scan(roots:)` deduplicates when roots overlap

**Status:**
- Vault has scope + marker present → `.liveHere`
- Vault has scope + no marker in scanRoots → `.liveElsewhere`
- Vault has no scope + marker exists → `.orphaned`

**Materialize (line-preservation is central):**
- Materialize into a nonexistent `.env` file → creates file with owned keys only
- Materialize into an `.env` file with only non-owned lines → preserves them, appends owned keys
- Materialize into an `.env` file with mixed owned + non-owned lines → replaces owned lines in place, preserves non-owned lines byte-for-byte
- Materialize with owned key already matching vault value → `.unchanged`
- Materialize with owned key differing → `.diffPending` when `overwriteDrift == false`
- Materialize with same diff + `overwriteDrift == true` → `.wrote`
- Line ordering preserved: existing owned key stays in its position; new owned keys append at the end
- Comment placement preserved: `# comment` before a rewritten owned key stays before it
- Trailing newline preserved: file ending with `\n` re-ends with `\n`; file ending without one same
- Canonical rewrite: value with spaces gets double-quoted output; simple value gets bare output
- Value with `#` gets double-quoted

**Clean:**
- Clean with no file → `.fileMissing`
- Clean with only owned keys → file deleted, `fileStillExists: false`
- Clean with owned + non-owned → file kept with non-owned lines only
- Clean removes trailing blank lines that were between owned keys (natural consequence of filtering)

**Heal:**
- Heal on missing file → all owned keys reported as `.fileMissing`
- Heal on file with all owned keys matching → all `.match`
- Heal on file with one drifted owned key → one `.fileValueDiffers` with both SHAs
- Heal SHAs are lowercase hex, 64 chars
- Heal never surfaces non-owned lines in the report

**Round-trip integrity (integration-style):**
- Create a vault with two owned keys, project dir with `.env` containing those two + three non-owned lines and two comments
- Materialize → file rewritten with vault values for owned keys, non-owned lines byte-for-byte preserved, comments preserved
- Delete one owned key from vault (via VaultCore) → next materialize should stop writing that line; the file's now-stale line becomes non-owned from the Materializer's perspective (which is correct — the key isn't owned anymore)

## Stop Condition

**If `VaultCore` does not expose "list owned keys for a scope" or "get value by scope+key" cleanly**, stop and surface what the current API looks like. The Materializer needs both. If the current API forces the Materializer to walk `<KEY>.age` / `<KEY>.link` files directly, that's a design leak we should decide about — either extend VaultCore, or accept the leak. Do not silently reimplement vault-file scanning inside the Materializer.

**If the line-preservation round-trip test cannot pass byte-for-byte because the parser drops information** (trailing whitespace, quoting variants, comments-after-values), stop and surface which cases fail. The kamae-2.2 contract is byte-for-byte preservation of non-owned lines. Weakening it needs a decision — not a silent workaround.

**If `URL(fileURLWithPath:relativeTo:).standardizedFileURL` behaves surprisingly on Linux** (Foundation's URL behavior is not fully identical across platforms), stop and surface the discrepancy. Marker portability is a stated property; a per-platform difference is a bug.
