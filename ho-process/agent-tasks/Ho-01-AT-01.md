# Ho-01-AT-01 — Filesystem primitives and pure-filesystem vault operations

## Goal

Implement the `SharibakoCore` filesystem layer: the `Shell.run()` internal utility, `VaultError` enum, vault directory layout helpers, Yams + Codable models, and the five vault operations that don't require `age` decryption — `listScopes`, `listShared`, `getScope`, `inspect`, `link` — plus link graph resolution and orphan detection. Full unit test coverage using ephemeral temp directories. No `age` binary needed to run these tests.

## Context

Read before implementing:

- `ho-process/hos/ho-01-vault-core.md` (parent ho — Think phase decisions, especially Decisions 2–4)
- `ho-process/kamae-2-sharibako-system-design.md` §2 (Vault Core) and §4 (Data Model)
- `CLAUDE.md` (project conventions: filesystem IS the schema, `age` shelled out via `Process`)

Vault layout from the system design:

```
vault/
├── .git/
├── shared/
│   └── <shared-id>.age         ← encrypted content, AT-02 handles
└── scopes/
    └── <scope-id>/
        ├── scope.yaml           ← plaintext metadata
        ├── <KEY>.age            ← encrypted content, AT-02 handles
        └── <KEY>.link           ← plaintext shared entry ID
```

This task implements everything in that layout except the `.age` encryption/decryption.

### 2. Shell.swift — internal subprocess utility

`internal` access only — not part of `SharibakoCore`'s public API.

- `ShellResult`: internal struct with `exitCode: Int32`, `stdout: String`, `stderr: String`.
- `Shell.findExecutable(_ name: String) throws -> URL`: probes these paths in order — `/opt/homebrew/bin/<name>`, `/usr/local/bin/<name>`, `/home/linuxbrew/.linuxbrew/bin/<name>`, `/usr/bin/<name>` — and returns the first URL where the file exists. Throws `VaultError.shellNotFound(name: name)` if none found.
- `Shell.run(_ executableURL: URL, _ arguments: [String]) throws -> ShellResult`: configures a `Foundation.Process` with `Pipe` on stdout and stderr, launches it, waits for exit, and returns a `ShellResult`. Throws on `Process.run()` failure (wraps as `VaultError.fileSystemError` or lets the Swift error propagate — caller handles). Does not throw on non-zero exit codes; callers check `exitCode`.

### 7. VaultCore.swift — main type with AT-01 operations

Public `struct VaultCore`. AT-02 will add a second initializer and four more methods; keep the type open for extension.

**Initializer:**

`public init(vaultURL: URL) throws` — stores the URL. Verifies the directory exists using `FileManager.default.fileExists(atPath:)`. Throws `VaultError.vaultNotFound(path: vaultURL)` if absent. Does not create the vault — call `VaultLayout.createVaultLayout(at:)` separately when initializing a new vault.

**Methods:**

- `public func listScopes() throws -> [ScopeMetadata]` — reads `scopes/` directory, for each subdirectory that contains a `scope.yaml` decodes it via `YAMLDecoder().decode(ScopeMetadata.self, from: String(contentsOf:))`. Returns sorted by `identity`. Throws `VaultError.fileSystemError` on directory read failure; throws `VaultError.yamlDecodeError` if any `scope.yaml` fails to decode.

- `public func listShared() throws -> [String]` — reads `shared/` directory, returns the `lastPathComponent` stem (without `.age` extension) of each `.age` file, sorted alphabetically.

- `public func getScope(_ id: String) throws -> ScopeMetadata` — reads and decodes `scopes/<id>/scope.yaml`. Throws `VaultError.scopeNotFound(id:)` if the directory or YAML is absent.

- `public func inspect(_ scopeID: String) throws -> [SecretInfo]` — reads `scopes/<scopeID>/`, returns one `SecretInfo` for each `.age` and `.link` file (excludes `scope.yaml`). For `.link` files: reads the file content as a UTF-8 string, trims whitespace, uses it as the `sharedID`. For `.age` files: `SecretInfo(key: stem, kind: .value)`. Throws `VaultError.scopeNotFound(id:)` if the scope directory is absent. Returns sorted by `key`.

- `public func link(_ key: String, inScope scopeID: String, toShared sharedID: String) throws` — writes the string `sharedID` to `scopes/<scopeID>/<key>.link` (UTF-8, no trailing newline). If `scopes/<scopeID>/<key>.age` exists, deletes it. Throws `VaultError.scopeNotFound(id:)` if the scope directory is absent.

- `public func linkGraph() throws -> [String: [(scopeID: String, key: String)]]` — walks all `scopes/*/*.link` files. For each, reads the shared entry ID and records `(scopeID: directoryName, key: fileStem)` under that ID in the result dictionary. Returns the full graph.

- `public func orphanedSharedEntries() throws -> [String]` — returns shared entry IDs present in `shared/` that have no entries in `linkGraph()`. Sorted.

### 9. VaultCoreFilesystemTests.swift — test suite

swift-testing `@Suite("VaultCore Filesystem")` with `@Test` functions. Each test creates a fresh temp directory:

```swift
let tempDir = FileManager.default.temporaryDirectory.appendingPathComponent(UUID().uuidString)
try FileManager.default.createDirectory(at: tempDir, withIntermediateDirectories: true)
defer { try? FileManager.default.removeItem(at: tempDir) }
```

Tests to cover:

- `VaultCore.init` succeeds for an existing directory, throws `vaultNotFound` for a nonexistent one
- `listScopes` on a vault with zero scopes returns empty array
- `listScopes` on a vault with three scopes returns all three sorted by identity
- `listScopes` throws `yamlDecodeError` if a `scope.yaml` is malformed
- `listShared` on a vault with zero shared entries returns empty array
- `listShared` on a vault with two shared entries returns both stems sorted
- `getScope` returns the decoded metadata for a valid scope
- `getScope` throws `scopeNotFound` for an absent scope
- `inspect` on a scope with only `.age` files returns all as `.value` kind
- `inspect` on a scope with only `.link` files returns all as `.link(sharedID:)` kind with correct IDs
- `inspect` on a mixed scope returns both kinds correctly
- `inspect` excludes `scope.yaml` from results
- `link` creates the `.link` file with correct content
- `link` deletes a pre-existing `.age` file for the same key
- `link` throws `scopeNotFound` for an absent scope
- `linkGraph` builds the correct mapping across multiple scopes and shared entries
- `orphanedSharedEntries` identifies a shared entry with no referencing links
- `orphanedSharedEntries` returns empty when all shared entries are referenced

Test fixture helpers (within the test file, not a separate file):

- `makeScopeYAML(identity: String, type: ScopeType) -> String` — returns a minimal `scope.yaml` YAML string
- `writeScope(_ id: String, type: ScopeType, in vault: URL) throws` — creates `scopes/<id>/scope.yaml`
- `writeLink(_ key: String, inScope: String, sharedID: String, in vault: URL) throws`
- `writePlaceholderAge(_ key: String, inScope: String, in vault: URL) throws` — writes an empty or short byte sequence as a stand-in `.age` file (AT-01 tests don't decrypt)
- `writeSharedPlaceholderAge(_ id: String, in vault: URL) throws`
