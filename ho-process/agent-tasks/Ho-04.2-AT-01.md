# Ho-04.2-AT-01 — Interactive select widget + ingest decision-source interface

## Goal

Build the reusable terminal-UX machinery `sharibako init` needs, standalone and fully tested, before the `init` command exists. Two pieces:

1. `InteractiveSelect` — a raw-mode arrow-key single-select prompt (↑/↓ + Enter over a highlighted choice list), with ANSI redraw, SIGINT-safe terminal restore, and non-TTY detection.
2. `IngestDecisionSource` — a protocol that turns a `ProposedScope` into `[KeyDecision]`, with an interactive TTY implementation that drives `InteractiveSelect` per detected key, and a scripted implementation used by tests (and, later, by the deferred non-interactive `init` path).

The seam is deliberate (ho-04.2 Decision 1): all decision *logic* lives behind `IngestDecisionSource` and is tested through a scripted double; the raw-mode renderer is thin and coverage-excluded. Nothing in this task imports or references an `InitCommand` — AT-02 wires it.

## Required Changes

### 1. `TerminalDetector.isInteractiveInput`

```swift

static var isInteractiveInput: Bool { isatty(STDIN_FILENO) != 0 }
```

### 2. `InteractiveSelect.swift`

A single-select arrow-key prompt over a list of string choices. Design for testability the same way `ValueInput` and `CleanCommand` do — inject the byte source and the output sink; default them to the real terminal.

```swift
struct InteractiveSelect {
    let title: String
    let choices: [String]
    var initialIndex: Int = 0

    /// Byte source: reads one input byte at a time. Defaults to raw-mode stdin.
    var readByte: () -> UInt8? = { /* raw-mode single-byte read */ }
    /// Output sink for the redraw. Defaults to stderr (prompts are UX, not payload).
    var write: (String) -> Void = { FileHandle.standardError.write(Data($0.utf8)) }
    /// TTY guard. Defaults to TerminalDetector.isInteractiveInput.
    var isInteractive: () -> Bool = { TerminalDetector.isInteractiveInput }

    /// Returns the chosen index, or throws CLIError.notInteractiveTerminal when
    /// input is not a TTY (do NOT block on a raw read that will never arrive).
    func run() throws -> Int
}
```

Behavior:
- On entry, if `isInteractive()` is false, throw `CLIError.notInteractiveTerminal` immediately.
- Enter raw mode via `termios` (`tcgetattr`/`tcsetattr`, clear `ICANON` + `ECHO`); **always** restore the saved termios in a `defer`, and install a SIGINT handler (or restore-then-reraise) so Ctrl-C leaves the terminal cooked, not mangled.
- Redraw: render `title`, then each choice with the highlighted row marked (ANSI reverse or a `>` cursor + bold; match `OutputRenderer` color discipline; honor no-color when appropriate). Repaint in place by moving the cursor up `choices.count` lines between redraws — do not scroll the scrollback with a fresh paint each keypress.
- Keys: `↑`/`↓` (escape sequences `ESC [ A` / `ESC [ B`) move the cursor with clamping (no wrap, or wrap — pick one and note it; clamping is fine); `Enter` (`\n`/`\r`) confirms and returns the index; `Ctrl-C` (`0x03`) restores and throws `CLIError.aborted`. Ignore other bytes.
- The raw-mode `readByte` default and the termios/redraw code are the coverage-excluded surface — mark with `// swiftlint:disable` / coverage-exclusion comments naming why (no headless TTY in CI). The *selection state machine* (cursor movement, key dispatch, clamping, return value) must be reachable and tested through injected `readByte`/`write`.

Refactor the state machine so tests drive it without a real terminal: feed a scripted `readByte` sequence (e.g. down, down, up, enter) and assert the returned index and the sequence of rendered frames captured through `write`.

### 3. `IngestDecisionSource.swift`

```swift
/// Produces a per-key decision for every DetectedKey in a proposal.
protocol IngestDecisionSource {
    /// - Parameter proposal: the scope proposal from Materializer.ingest.
    /// - Parameter sharedIDs: existing shared-entry IDs (for the link-to-shared choice).
    /// - Returns: one KeyDecision per detected key, in proposal order.
    func decisions(for proposal: ProposedScope, sharedIDs: [String]) throws -> [KeyDecision]
}
```

The five choices per key map to `KeyDecision`:
- import as scope-local → `.importAsLocal(key:)`
- link to shared → `.linkToShared(key:sharedID:)` (needs a second selection: which shared ID)
- move to shared → `.moveToShared(key:newSharedID:)` (needs a second input: the new slug)
- leave alone → `.leaveAlone(key:)`
- skip → `.skip(key:)`

### 4. `InteractiveIngestPrompt.swift` (TTY implementation)

`struct InteractiveIngestPrompt: IngestDecisionSource`.

- Print a header naming the scope and detected-key count before the walk.
- For each `DetectedKey`, run an `InteractiveSelect` over the five labels. When `detected.nameMatchedSharedID != nil`, set `initialIndex` to the "link to shared" row and include the matched ID in the label ("link to shared (`openai-personal`)").
- On "link to shared": if `sharedIDs` is empty, that choice is unavailable (omit it or show it disabled with a note); otherwise run a second `InteractiveSelect` over `sharedIDs`. Default the cursor to `nameMatchedSharedID` when present.
- On "move to shared": prompt for a new slug (line input via an injected reader, same shape as `ValueInput`); sanitize with the same rules the scope ID uses. (Reuse or factor the sanitizer if `Materializer` doesn't expose one publicly — a small local sanitize is acceptable; do not modify `SharibakoCore`.)
- Inject the underlying `InteractiveSelect` factory and line-reader so this type is testable without a terminal, OR keep the type a thin composition and cover the branching through `ScriptedIngestDecisionSource` — your call, but every branch (matched/unmatched, empty/non-empty sharedIDs, each of the five choices) must be covered.

### 5. `ScriptedIngestDecisionSource.swift`

A deterministic `IngestDecisionSource` that returns pre-set decisions. Two constructors worth having:
- explicit `[String: KeyDecision]` keyed by key name (throw or default for a key not covered);
- a uniform "same decision for every key" convenience (e.g. `.allImportLocal`), which the deferred non-interactive `--import-all` path will lean on later.

This lives in `Sources` (not `Tests`) precisely so AT-02's tests and the future non-interactive path share it.

### 6. `CLIError` additions

Add cases (in the existing `CLIError` enum): `notInteractiveTerminal`, `aborted`. Map them to messages + exit codes via the existing `ErrorReporter`/exit-code taxonomy (`notInteractiveTerminal` → user-error class 2 with "`init` needs an interactive terminal; scriptable flags are a followup"; `aborted` → a clean cancel, exit 0 or the project's cancel convention — match how `GenerateCommand` treats "Aborted.").

### 7. Tests

`InteractiveSelectTests.swift`:
- Scripted `readByte` sequences: down→enter returns index 1; down→down→up→enter returns index 1; enter-at-top returns 0; clamping at both ends; `0x03` throws `CLIError.aborted`.
- `isInteractive` false → `run()` throws `notInteractiveTerminal` without consuming bytes.
- Rendered frames captured via injected `write` assert the highlight moves with the cursor.

`IngestDecisionSourceTests.swift`:
- `ScriptedIngestDecisionSource` returns the mapped decision per key; `.allImportLocal` maps every detected key to `.importAsLocal`.
- `InteractiveIngestPrompt` driven with an injected select-factory: a `nameMatchedSharedID` key defaults to link-to-shared and yields `.linkToShared` with the matched ID; empty `sharedIDs` removes/disables the link choice; "move to shared" collects and sanitizes a new slug into `.moveToShared`; the other three choices yield their decisions.

## Do Not

- Do not modify `SharibakoCore` (`ProposedScope`, `DetectedKey`, `KeyDecision`, `Materializer`). Consume the ho-03 surface as-is. If something is genuinely missing, stop and surface it.
- Do not build `InitCommand` or register anything in `SharibakoCommand` — that's AT-02.

- Do not block on a raw-mode read when stdin is not a TTY — detect and throw `notInteractiveTerminal` first.
- Do not build the non-interactive `--import-all` flag surface (deferred, Decision 6). `ScriptedIngestDecisionSource` is the seam it will use later; that's all this task provides toward it.

## Stop Condition

**If the termios raw-mode restore cannot be made SIGINT-safe** within the project's structure (e.g. the signal handler can't reach the saved termios cleanly), stop and surface — a prompt that leaves the terminal in raw mode on Ctrl-C is not shippable, and the fix might want a small shared terminal-state holder rather than a per-`InteractiveSelect` handler.

**If `Materializer` exposes no reusable slug sanitizer** and duplicating the sanitize logic here risks drift from the scope-ID rules, stop and surface — the right move might be a small `public` sanitize on `SharibakoCore`, which is a library change this task otherwise forbids.
