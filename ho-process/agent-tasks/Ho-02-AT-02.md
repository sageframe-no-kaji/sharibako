### 1. Conduit+Remote.swift — extension with network operations

Follow the pattern of `VaultCore+Encryption.swift`: separate file, extension on the type, keeps the main file under the 300-line body ceiling.

**`public func push() throws -> PushResult`**

1. Call `remoteURL()`. If it returns `nil`, return `.noRemote`.
2. Run `git push origin HEAD` via the internal `git(_:)` helper.
3. On exit 0: parse stdout to determine how many commits were pushed. If "Everything up-to-date" appears in stderr, return `.upToDate`. Otherwise return `.success(commitsPushed: N)` — count commits from stdout, or fall back to counting via `git rev-list @{u}..HEAD` before the push (capture before push, count is what got pushed).
4. On nonzero exit with "non-fast-forward" or "rejected" in stderr: return `.rejected(reason: <trimmed stderr>)`.
5. On other nonzero exit: throw `VaultError.gitInvocationFailed(exitCode:, stderr:)`.

**`public func pull() throws -> PullResult`**

1. Call `remoteURL()`. If `nil`, return `.noRemote`.
2. Run `git fetch origin` first to update remote-tracking refs. If that fails, throw `gitInvocationFailed`.
3. Check `git rev-list --left-right --count HEAD...@{upstream}` — if the remote count (right side) is 0, return `.upToDate`.
4. Run `git pull --no-rebase --no-ff origin` (explicit merge strategy).
5. On exit 0: parse "Fast-forward" or merge commit output to count commits pulled. Return `.success(commitsPulled: N)`.
6. On nonzero exit with "CONFLICT" markers in stderr or stdout: this is a merge conflict.
   a. Parse the conflicting file paths from git's output (`CONFLICT (content): Merge conflict in <path>` lines).
   b. For each conflicting path, get the local and remote SHAs via `git ls-files --stage <path>` (stage 2 is local/ours, stage 3 is remote/theirs).
   c. Run `git merge --abort` to restore clean state.
   d. Build `[ConflictedFile]` array with absolute paths (join relative path with `vaultURL`), local + remote SHAs.
   e. Return `.abortedConflict(conflicts: [...])`.
7. On other nonzero exit: throw `VaultError.gitInvocationFailed`.

The exact regex / string-parsing patterns for git output are deferred to execution — the shape is fixed, the specific pattern is adjusted against actual git output during test writing.

### 3. Tests/…/ConduitRemoteTests.swift — test suite

swift-testing `@Suite("Conduit Remote")`. Cover:

- `push` with no remote configured returns `.noRemote`
- `pull` with no remote configured returns `.noRemote`
- `push` when already up-to-date returns `.upToDate`
- `pull` when already up-to-date returns `.upToDate`
- **Bare-remote round trip**: in vaultA, `VaultCore(...).addSecret(...)`; `commit(message:)`; `push()` returns `.success`; in vaultB, `pull()` returns `.success`; the secret decrypts to the same value in vaultB.
- **Push rejected by non-fast-forward**: vaultA and vaultB both commit + push in parallel; the second push returns `.rejected(reason:)`.
- **Auto-aborted conflict**: vaultA rotates `OPENAI_API_KEY`, commits, pushes. vaultB rotates the same key to a different value, commits, then pulls. `pull()` returns `.abortedConflict(conflicts: [...])`, the conflict list names the specific file, and `Conduit(vaultB).status()` afterwards shows `dirty: false` (clean state — merge was aborted).
- `ConflictedFile.localSHA` and `.remoteSHA` are non-empty and both retrievable via `git cat-file -p <sha>` (verify by running a subprocess).

## Do Not

- Do not add UI or CLI code — surfaces are ho-04 and later.
- Do not attempt to auto-resolve conflicts. The Conduit's job stops at aborting and returning structured conflict info; the resolver UX is a ho-04 followup tracked in the parent ho document.
- Do not implement remote authentication (SSH agent, credential helper). The Conduit assumes the user's git config already works. Tests use a local file-path remote (`git init --bare` in a temp directory), which needs no auth.
- Do not use `git pull --rebase` or `git pull --ff-only`. Explicit `--no-rebase --no-ff` — the auto-abort code path assumes a merge, and a rebase conflict aborts differently (`git rebase --abort` instead of `git merge --abort`).
- Do not add new cases to `VaultError`. All network states are enum return values per Decision 6.
- Do not shell out to `git` outside the `Conduit` type. Every git call goes through the internal `git(_:)` helper defined in AT-01.

## Stop Condition

**If `git pull --no-rebase --no-ff` produces conflict output in a shape substantially different from the pattern documented** (e.g. localization changing `CONFLICT (content): Merge conflict in <path>` into something else, or the file paths in stderr are relative to a different root than the vault), stop and surface the actual output before adapting the parser.

**If `git ls-files --stage <path>` doesn't produce entries for stages 2 and 3 after a conflict** (e.g. the conflict is a delete-vs-modify with different stage numbers), stop and surface the actual stage entries. Adapt `ConflictedFile` construction based on the real output.

**If the round-trip fixture flakes** (e.g. concurrent pushes interfere in ways that are hard to reproduce), stop and surface the flake pattern. Reliability of the fixture is a prerequisite; don't paper over intermittent failures.
