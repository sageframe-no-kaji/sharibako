# Ho-04.4-AT-01 — `IngestDashboard` widget + dashboard decision source

## Goal

Build the reusable terminal-UX machinery the new `sharibako init` needs, standalone and fully tested, before rewiring the command. Two pieces:

1. `IngestDashboard` — a full-screen, alternate-screen raw-mode widget that shows every detected key as one row, lets the user set each key's routing choice in place, mark rows for bulk, apply a bulk action to the marked-or-all set, collect the secondary inputs that link and move need, and commit or cancel the whole screen at once.
2. `DashboardIngestPrompt` — an `IngestDecisionSource` implementation that builds the dashboard from a `ProposedScope` + shared IDs, runs it, and maps the result to `[KeyDecision]`.

The raw-mode renderer (byte reads, alternate-screen escapes, `termios`) is coverage-excluded; the decision *state machine* (navigation, cycle, mark, ALL, secondary-input resolution, final map to `[KeyDecision]`) is driven through injected `readByte`/`write` and fully covered. Nothing in this task touches `InitCommand` — AT-02 rewires it.

## Required Changes

### 1. `IngestDashboard.swift` — the widget

A full-screen single-run widget over a set of key rows. Design for testability exactly the way `InteractiveSelect` does — inject the byte source, the output sink, and the TTY guard; default them to the real terminal.

Shape (adapt names/structure as cleaner constructions suggest; these are the required behaviours, not a prescribed API):

```swift
struct IngestDashboard {
    /// One row: a key and the actions it may take. `sharedIDs` is the pool the
    /// link sub-pick draws from. Rows start on their suggested choice.
    struct Row { let key: String; let matchedSharedID: String?; /* current choice state */ }

    let rows: [Row]
    let sharedIDs: [String]
    let banner: [String]              // read-only context lines (needs-values, parse warnings)

    var readByte: () -> UInt8? = { /* raw-mode single-byte read — coverage-excluded */ }
    var write: (String) -> Void = { fputs($0, stderr) }
    var isInteractive: () -> Bool = { TerminalDetector.isInteractiveInput }

    /// Runs the dashboard. Returns the resolved choice per row (in `rows` order)
    /// on commit; throws `CLIError.aborted` on q/Esc cancel and
    /// `CLIError.notInteractiveTerminal` when input is not a TTY.
    func run() throws -> [ResolvedChoice]   // ResolvedChoice carries enough to build a KeyDecision
}
```

Behaviours (all per parent ho Decisions 1–6):

- **Non-TTY guard.** On entry, if `isInteractive()` is false, throw `CLIError.notInteractiveTerminal` immediately — never block on a raw read that will not arrive.
- **Alternate screen (Decision 3).** Enter the alternate screen buffer on start (`ESC [ ? 1049 h` or the terminfo `smcup` equivalent) and leave it on exit (`ESC [ ? 1049 l` / `rmcup`). Reuse `InteractiveSelect`'s `termios` raw-mode entry and its module-level saved-state SIGINT handler; extend the restore so Ctrl-C also leaves the alternate screen — an interrupt must return a cooked terminal on the original screen. Always restore in a `defer`.
- **Layout (Decision 1).** Render the banner (read-only context), then one row per key with spacing between rows for legibility. Each row shows the key name and its current choice — import / link / move / leave / skip — as a colour change plus a `✓`, NOT a reverse-video block. Mark the active row distinctly (cursor glyph and/or highlight). Show marked rows (Decision 2) with a visible marker. Render a short key legend/footer. Redraw cursor-addressed (position, clear, paint) each keystroke — do not append frames.
- **Navigation (Decision 1).** `↑`/`↓` (`ESC [ A`/`ESC [ B`) move the active row, clamping at both ends. `←`/`→` (`ESC [ D`/`ESC [ C`) cycle the active row's choice through its available actions in place.
- **Initial state (Decision 1).** A row whose `matchedSharedID != nil` starts on **link** with that entry pre-resolved and named in the row; every other row starts on **import**.
- **Mark + ALL (Decision 2).** `space` toggles the active row's mark. `a` invokes ALL: pick one of the target-free actions — **import** / **leave** / **skip** — and apply it to every marked row, or to every row when none are marked. ALL overwrites the affected rows' current choices. Link and move are NOT offered in ALL (they need per-key targets).
- **Secondary inputs, in-dashboard (Decision 4).**
  - Cycling a row to **link** (or choosing it) resolves a shared entry: if `sharedIDs` is empty, link is unavailable for that row (skip the choice in the cycle); otherwise open an in-dashboard sub-list over `sharedIDs`, navigated like the main list, defaulting to `matchedSharedID` when present. The resolved ID shows in the row.
  - Cycling a row to **move** opens an in-dashboard single-line slug field driven in raw mode: printable ASCII appends, backspace (`0x7F`/`0x08`) deletes the last char, Enter confirms, Esc cancels back to the prior choice. Sanitize the confirmed slug with the same rules the scope ID uses (see change 3). No `readLine()`; no dropping out of raw mode; no leaked escape bytes.
- **Commit / cancel (Decision 6).** `Enter` (`\n`/`\r`) commits — return the resolved choices for all rows. `q` (`0x71`) or `Esc` (a lone `0x1B` not starting an escape sequence) cancels — throw `CLIError.aborted`. `Ctrl-C` (`0x03`) also cancels via the SIGINT path. Ignore other bytes.
- **Coverage exclusion.** The default `readByte` closure body and the `termios`/alternate-screen/SIGINT code are coverage-excluded — mark with `// swiftlint:disable` / a coverage-exclusion comment naming why (no headless TTY in CI). Everything else — cursor movement, cycle, clamp, mark, ALL application, sub-list resolution, slug field editing, the returned result — must be reachable through injected `readByte`/`write`.

Structure the state machine so tests drive it without a terminal: feed a scripted `readByte` sequence and assert both the returned `[ResolvedChoice]` and, where it matters, the rendered frames captured through `write`.

### 2. `DashboardIngestPrompt.swift` — the `IngestDecisionSource`

```swift
struct DashboardIngestPrompt: IngestDecisionSource {
    func decisions(for proposal: ProposedScope, sharedIDs: [String]) throws -> [KeyDecision]
}
```

- Build the dashboard rows from `proposal.detectedKeys` (carry `key` and `nameMatchedSharedID`), pass `sharedIDs`, and pass a banner built from `proposal.suggestedKeysNeedingValues` and `proposal.parseWarnings`.
- Run the dashboard; map each `ResolvedChoice` to its `KeyDecision` in `proposal.detectedKeys` order: import → `.importAsLocal(key:)`, link → `.linkToShared(key:sharedID:)`, move → `.moveToShared(key:newSharedID:)`, leave → `.leaveAlone(key:)`, skip → `.skip(key:)`.
- Inject the dashboard runner (a factory returning `[ResolvedChoice]` from the built rows) so this type's mapping is testable without a terminal, mirroring how `InteractiveIngestPrompt` injects `selectFactory`. Propagate `CLIError.aborted`/`CLIError.notInteractiveTerminal` unchanged.

### 3. Slug sanitizer

Reuse the same sanitize rules the scope ID uses. `InteractiveIngestPrompt.swift` already has a `private func sanitizeSharedID(_:)` — that file is deleted in AT-02, so lift the function into a place the dashboard can use (a small `private`/`fileprivate` helper alongside `IngestDashboard`, or a tiny internal free function in `Support/`). Do not modify `SharibakoCore`. If duplicating the logic risks drift from the scope-ID rules, see the Stop Condition.

### 4. Tests

`IngestDashboardTests.swift` (swift-testing, injected `readByte`/`write`, scripted byte queues):
- Navigation: `↓` then Enter commits with the second row still on its default; `↓ ↓ ↑` lands on the second row.
- Cycle: `→`/`←` on a row moves its choice through the actions and back; committing returns the cycled choice.
- Initial state: a row with `matchedSharedID` starts on link with that ID; a row without starts on import.
- Mark + ALL: mark two rows with `space`, invoke `a`, pick import → only the marked rows become import; with nothing marked, `a` + import sets every row; ALL over leave/skip works; link/move are not selectable in the ALL pick.
- Link sub-pick: choosing link opens the shared list, selecting an entry records `.linkToShared` with that ID; empty `sharedIDs` makes link unavailable in the cycle.
- Move slug field: type characters, backspace, Enter → the sanitized slug is recorded; stray/escape bytes do not corrupt it (regression guard for the ho-04.2 bug); Esc in the field cancels back to the prior choice.
- Commit/cancel: Enter returns the resolved choices; `q` and lone `Esc` throw `CLIError.aborted`; `0x03` throws `aborted`.
- Non-TTY: `isInteractive` false → `run()` throws `notInteractiveTerminal` without consuming bytes.

`DashboardIngestPromptTests.swift`:
- With an injected dashboard runner returning canned resolved choices, assert the `[KeyDecision]` mapping is correct and in `detectedKeys` order for all five choices, including the matched-link and move-slug cases.
- Banner is built from `suggestedKeysNeedingValues` + `parseWarnings` (assert it reaches the dashboard rows/banner input).

## Do Not

- Do not modify or extend `InteractiveSelect` — it stays the single-question widget (Decision 5). The dashboard is a new widget; reuse the `termios`/SIGINT pattern by mirroring it, not by adding a table mode to `InteractiveSelect`.
- Do not modify `SharibakoCore` (`ProposedScope`, `DetectedKey`, `KeyDecision`, `Materializer`). Consume the ho-03 surface as-is. If something is genuinely missing, stop and surface it.
- Do not touch `InitCommand` or delete `InteractiveIngestPrompt` — that is AT-02.
- Do not offer link or move in the ALL bulk action — they need per-key targets (Decision 2).
- Do not collect the move slug or the link pick with `readLine()` or by leaving raw mode mid-run — collecting them in-dashboard is the whole point (Decision 4); a bare `readLine()` reintroduces the ho-04.2 corruption.
- Do not block on a raw read when stdin is not a TTY — detect and throw `notInteractiveTerminal` first.
- Do not add a package dependency for the TUI. Hand-roll on the existing `termios` pattern, consistent with ho-04.2.
- Do not build the non-interactive `--import-all` flag surface (still deferred, ho-04.2 Decision 6).

## Stop Condition

- **If the alternate-screen restore cannot be made SIGINT-safe** within the module-level saved-`termios` pattern (e.g. the handler can't cleanly leave the alternate screen as well as restore `termios`), stop and surface — a dashboard that leaves the terminal on the alternate screen or in raw mode after Ctrl-C is not shippable, and the fix may want a small shared terminal-state holder rather than bolting onto the existing one.
- **If lifting `sanitizeSharedID` out of the soon-deleted `InteractiveIngestPrompt` risks drift** from the scope-ID sanitize rules (they must stay identical), stop and surface — the right move might be a small `public` sanitizer on `SharibakoCore`, which this task otherwise forbids.
