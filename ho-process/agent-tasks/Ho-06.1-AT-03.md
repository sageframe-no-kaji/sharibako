# Ho-06.1-AT-03 — Reveal ergonomics + dialog windows

**Goal**

Cut reveal friction and make the dialogs honest citizens: the Keychain
provider shares one `LAContext` with a 5-minute Touch ID reuse window; a
"Preview .env" action renders exactly what Materialize would write (via a new
`Materializer.preview(marker:)`) behind one Touch ID; secure fields get a
show-while-typing toggle; rotate prefills only when the value is already
revealed; and the three Add sheets become movable, non-modal windows. Closes
the ho with the signed-install + Touch-ID dogfood gate.

**Context**

ho-05 gate findings: every operation re-prompted Touch ID (tolerable in the
CLI, grating in a windowed app); modal sheets blocked checking an existing
secret mid-add. Decisions 5 and 6-windows in
`ho-process/hos/ho-06.1-workshop-responsiveness.md` — including the ratified
security read on the 5-minute window (exposure is an unlocked unattended Mac
only; locking gates re-entry; software-class threats unchanged). The reuse
window applies to all key loads — reveal, add, rotate, materialize — they load
the same key.

**Required Changes**

1. **Shared `LAContext` in `GUIKeychainAgeKeyProvider`.** Replace the
   per-operation `LAContext()` with a shared, cached context whose
   `touchIDAuthenticationAllowableReuseDuration` is
   `LATouchIDAuthenticationMaximumAllowableReuseDuration` (5 minutes).
   Repeated key loads inside the window must not re-prompt. Handle
   invalidation: a cancelled/failed evaluation may invalidate the context —
   recreate it on the next load rather than failing permanently. The Keychain
   query itself (service, account, access group) is untouched.

2. **`Materializer.preview(marker:) -> String`** in Core. Returns the exact
   composed `.env` content `materialize` would write for the marker, without
   writing anything. Reuse the private composer — do not duplicate the merge
   logic. Unit tests: preview output equals the file `materialize` writes for
   the same fixture (byte-for-byte); preview creates/modifies no file; the
   drift case previews the vault-side composition.

3. **"Preview .env" action.** A model intent that acquires the age key (one
   Touch ID, riding the reuse window), resolves the marker from the AT-01
   cache, and calls `preview` through the `VaultWorker` (it decrypts every
   scope secret — same weight as materialize). `EnvPreviewSheet` renders the
   result: monospaced, text-selectable, titled with the target path ("This is
   what Materialize writes to <path>"). Entry point adjacent to Materialize
   (toolbar or detail-pane action; executor's call), disabled without a
   selected scope + cached marker.

4. **Secure-field eye toggle.** Add Secret, Add Shared Entry, and the rotate
   field get a show-while-typing toggle (SecureField ↔ TextField swap
   preserving focus/value). Default masked.

5. **Rotate prefill rule.** The edit-value field prefills the current value
   only when `revealedValue` is non-nil for that key; otherwise it starts
   empty. An unrevealed value must never be decrypted just to prefill a
   field. Unit-test the rule in the model tests if the prefill decision lives
   in the model; otherwise it's a view-layer conditional feeding from
   published state.

6. **Add dialogs become auxiliary windows.** The three Add forms open via
   `openWindow` as real SwiftUI `Window` scenes: movable, non-modal, main
   window stays interactive. `WorkshopModel` injected via `.environment` at
   the scene level. Add Secret receives the target scope (window value or
   read-at-open from selection — executor's call; it must not break when the
   selection changes mid-add). Windows close on successful submit; the
   creation announce (AT-02) fires in the main window's status surface.

7. **Dogfood gate (closes the ho).** Clean release build, signed install via
   `scripts/install.sh` (verify the installed binary is fresh — the `~/.local`
   path-shadow gotcha), real vault, real Keychain, production scan root:
   window interactive during launch scan / materialize / sync; two reveals
   inside five minutes cost one Touch ID and a lapsed window re-prompts;
   footer names vault + `sageframe-irori/sharibako-vault`; jump opens Finder
   at the marker; Preview .env matches `sharibako materialize` output and
   writes nothing; an Add window moves with the main window live behind it;
   creations announce with the pulse. The gate covers AT-01 and AT-02's
   surfaces too — it is the ho's close, not just this task's.

**Do Not**

- Do not weaken the Keychain query (service, account, access group,
  authentication requirement) to make reuse work — the reuse window lives in
  the `LAContext`, not in the query.
- Do not duplicate the merge/composition logic for preview — reuse the
  private composer.
- Do not build inline reveal-all in the secret list — deferred until real use
  asks (Decision 5b).
- Do not add an auto-hide timer or clipboard-clear to reveal — real use has
  not asked; out of scope for this ho.

**Stop Condition**

If the shared `LAContext` misbehaves at the gate in ways recreation-on-failure
doesn't fix (reuse window not honored, spurious invalidations, prompts on
every load), stop and surface with the observed behavior — do not ship a
loosened security posture or an undocumented workaround. The 5-minute window
is ratified; how it degrades is a practitioner decision.
