### 3. Conduit.swift — main type + local operations

Public `struct Conduit`. Mirrors `VaultCore`'s shape.

**Initializer:**

`public init(vaultURL: URL) throws` — verifies the vault directory exists (same check as `VaultCore.init`). Throws `VaultError.vaultNotFound(path:)` if not. Does NOT verify it's a git repo — that surfaces on first git operation.

**Methods:**

- `public func initializeRepository() throws` — if `<vault>/.git/` doesn't exist, runs `git init` in the vault directory. Idempotent (no-op if `.git/` exists). Throws `VaultError.gitInvocationFailed` on failure.

- `public func setIdentity(name: String, email: String) throws` — runs `git config --local user.name "<name>"` and `git config --local user.email "<email>"` inside the vault. Two separate `Shell.run` invocations. Both must succeed; a failure on the second still leaves the first written (acceptable).

- `public func setRemote(_ url: String) throws` — sets the `origin` remote. First runs `git remote get-url origin`; if that succeeds, runs `git remote set-url origin <url>`; if it fails (no origin), runs `git remote add origin <url>`.

- `public func remoteURL() throws -> String?` — runs `git remote get-url origin`. Returns the trimmed stdout on success, `nil` on nonzero exit (which is git's way of saying "no such remote"). Throws only if the shell call itself fails (e.g. `git` not on PATH).

- `public func status() throws -> StatusResult` — runs `git status --porcelain=v2 --branch`. Parse the output:
  - Header lines start with `# branch.oob <upstream>`, `# branch.ab +<ahead> -<behind>`
  - File lines: `1 <XY> ...` (tracked file changes), `2 <XY> ...` (renames/copies), `? <path>` (untracked)
  - `hasRemote` is derived from whether the `branch.oob` header names an upstream
  - `dirty` is `true` if any modified, deleted, or untracked files exist
  - Absolute paths are constructed by joining the parsed path with `vaultURL`
  If the parse fails or git is not initialized, throws `VaultError.gitInvocationFailed`.

- `public func commit(message: String) throws -> CommitResult` — runs `git add -A` at the vault root (via `Shell.run` with the working directory set — see below), then `git commit -m "<message>"`.
  - If `git commit` prints "nothing to commit" (exit code 1 with specific stderr pattern), return `.nothingToCommit`
  - Otherwise on success, parse the commit SHA from `git rev-parse HEAD` and return `.success(sha: ...)`
  - On any other failure, throw `VaultError.gitInvocationFailed`

**Working directory:**

`Shell.run(_:_:)` currently doesn't set the working directory — the `Process` inherits the parent's `cwd`. Because all `git` invocations here need to run inside the vault directory, extend `Shell.run` with an optional `workingDirectory: URL?` parameter (default `nil`). If provided, set `process.currentDirectoryURL` before launch.

**Internal helper:**

Add a private helper on `Conduit`:

```swift
private func git(_ arguments: [String]) throws -> ShellResult {
    let binary = try Shell.findExecutable("git")
    return try Shell.run(binary, arguments, workingDirectory: vaultURL)
}
```

Every local operation goes through this. Non-zero exits are the caller's job to interpret (some are informational — "no remote" — not errors).

### 6. Tests/…/ConduitLocalTests.swift — test suite

swift-testing `@Suite("Conduit Local")`. Cover:

- `init` succeeds for an existing vault directory
- `init` throws `vaultNotFound` for a nonexistent path
- `initializeRepository` creates `.git/` when absent
- `initializeRepository` is a no-op when `.git/` already exists
- `setIdentity` writes `user.name` and `user.email` into `.git/config` (verify by reading the file, or by running `git config --local --get user.name` via `Shell.run`)
- `setRemote` adds origin when there's no remote yet
- `setRemote` updates origin when one already exists
- `remoteURL` returns `nil` when no remote is configured
- `remoteURL` returns the configured URL when one is set
- `status` on an initialized empty repo returns `dirty=false`, `hasRemote=false`, `ahead=0`, `behind=0`
- `status` on a repo with a new file returns `dirty=true` with the file in `untrackedFiles`
- `status` on a repo with a modified tracked file returns `dirty=true` with the file in `modifiedFiles`
- `status` throws `gitInvocationFailed` for a directory that isn't a git repo
- `commit` with nothing to commit returns `.nothingToCommit`
- `commit` with staged changes returns `.success(sha:)` and the SHA matches `git rev-parse HEAD`
- `commit` on a repo without a configured identity throws `gitInvocationFailed` (verify git's default-identity refusal surfaces cleanly)

Test fixture pattern uses the new `withEphemeralGitVault` helper. For the "no identity configured" test, build a vault manually without calling `setIdentity`.

## Do Not

- Do not implement `push()` or `pull()` — those are AT-02.
- Do not shell out to `git` outside the `Conduit` type. `Shell.findExecutable("git")` is called only inside the `Conduit.git(_:)` helper.
- Do not add cases to `VaultError` beyond `gitInvocationFailed`. Per Decision 6, the network states (`noRemote`, `abortedConflict`, `rejected`) are enum return values, not errors.
- Do not read from or write to the user's global git config (`~/.gitconfig` or `~/.config/git/config`). Every `git config` call is `--local`.
- Do not detect SSH aliases, parse `~/.ssh/config`, or otherwise be clever about identity. That's a ho-04 followup.
- Do not add coverage-threshold gates in this task — CI already enforces the 90% floor.

## Stop Condition

**If `git status --porcelain=v2 --branch` output is meaningfully different from what's documented** (e.g. the runner has a git old enough to not support `--porcelain=v2`), stop and surface the actual output. Do not silently downgrade to v1 without noting the reason.

**If `setIdentity` on a fresh repo silently fails to make `git commit` accept a commit** (e.g. git wants a `user.signingkey` even without `--gpg-sign`), stop and surface what git actually complained about. The default expectation is that `user.name` + `user.email` at `--local` scope are sufficient.

**If `Shell.run` cannot be extended with a `workingDirectory` parameter without breaking existing callers** (`age`, `age-keygen`), stop and surface the conflict. The `age` callers were written without a working directory and should keep working with `workingDirectory: nil`.
