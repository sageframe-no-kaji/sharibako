**`age` invocation pattern:**

```bash
# Encrypt: extract public key from private key file header, write ciphertext to .age file
age --encrypt --recipient <public-key> -o <output.age> <plaintext-input-file>

# Decrypt: read ciphertext from .age file, write plaintext to stdout
age --decrypt --identity <private-key-file> <input.age>
```

The private key file generated by `age-keygen -o <file>` has this format:

```
# created: 2024-01-01T00:00:00+00:00
# public key: age1ql3z7hjy54pw3hyww5ayyfg7zqgvc7w3j2elw8zmrj2kg5sfn9aqmcac8p
AGE-SECRET-KEY-1...
```

Extract the public key by reading the second line and stripping the `# public key: ` prefix.

## Required Changes

### 1. Models/SecretContent.swift

Public `Codable` struct matching the decrypted `.age` content schema from the system design §4:

```swift
public struct SecretContent: Codable, Equatable {
    public let value: String
    public let notes: String?
    public let rotatedAt: String?

    enum CodingKeys: String, CodingKey {
        case value, notes
        case rotatedAt = "rotated_at"
    }
}
```

`rotatedAt` is an ISO 8601 date string (`"2026-06-30"`). Stored as a `String` in v1 — no `Date` conversion needed.

### 2. VaultCore.swift — second initializer

Add alongside the existing `init(vaultURL:)`:

```swift
public init(vaultURL: URL, ageKeyURL: URL) throws
```

Stores both URLs. Verifies `vaultURL` exists (same check as the existing initializer). Does not verify `ageKeyURL` at init time — missing key surfaces at first use as `VaultError.shellNotFound` or `VaultError.ageInvocationFailed`.

Extract and cache the age public key at init time: read the `ageKeyURL` file, find the line beginning with `# public key: `, strip the prefix, store as an internal `String`. This avoids re-reading the file on every `addSecret` or `rotate` call.

### 3. VaultCore.swift — encryption operations

Add four methods (all require `ageKeyURL` to be set; throw `VaultError.shellNotFound(name: "age")` if the key URL is nil — i.e., the caller used `init(vaultURL:)` instead of `init(vaultURL:ageKeyURL:)`):

---

**`public func addSecret(_ key: String, value: String, inScope scopeID: String, notes: String? = nil) throws`**

1. Encode `SecretContent(value: value, notes: notes, rotatedAt: today's ISO date)` to a YAML string via `YAMLEncoder()`.
2. Write the YAML to a temp file (`UUID().uuidString + ".yaml"` in `FileManager.default.temporaryDirectory`).
3. Run `age --encrypt --recipient <publicKey> -o <secretURL> <tempFile>` via `Shell.run(Shell.findExecutable("age"), [...])`.
4. Delete the temp file (in a `defer` block — runs even if step 3 throws).
5. Throw `VaultError.ageInvocationFailed(exitCode:stderr:)` if age exits non-zero.
6. Throw `VaultError.scopeNotFound(id:)` if the scope directory is absent.

---

**`public func getValue(_ key: String, inScope scopeID: String) throws -> String`**

1. Check for `<key>.link` — if present, read the shared entry ID and set the target file to `sharedEntryURL(id, in: vaultURL)`. If the shared `.age` file is absent, throw `VaultError.linkTargetMissing(id:)`.
2. If no `.link`, use `secretURL(key, inScope: scopeID, in: vaultURL)`. If absent, throw `VaultError.secretNotFound(scope: scopeID, key: key)`.
3. Run `age --decrypt --identity <ageKeyURL.path> <targetFile.path>` via `Shell.run`.
4. Throw `VaultError.ageInvocationFailed` if age exits non-zero.
5. Decode `result.stdout` as `SecretContent` via `YAMLDecoder`. Throw `VaultError.yamlDecodeError` if decoding fails.
6. Return `content.value`.

---

**`public func rotate(_ key: String, inScope scopeID: String, newValue: String) throws`**

1. Decrypt the existing `<KEY>.age` to get current `SecretContent` (for preserving `notes`). Throw `VaultError.secretNotFound` if absent.
2. Re-encode with updated `value` and `rotatedAt` (today's date); preserve existing `notes`.
3. Encrypt the new content to the same `<KEY>.age` path (overwrites).

**`public func rotateShared(_ sharedID: String, newValue: String) throws`**

Same pattern as `rotate` but operates on `vault/shared/<sharedID>.age`. Throw `VaultError.sharedEntryNotFound(id:)` if absent. Does not touch any `.link` files — links resolve at read time.

---

**`public func unlink(_ key: String, inScope scopeID: String) throws`**

1. Read `<KEY>.link` to find the `sharedID`. Throw `VaultError.secretNotFound(scope:key:)` if the `.link` is absent.
2. Decrypt the shared entry to get its current value via `getValue(key, inScope: scopeID)` — the link resolution in `getValue` handles this.
3. Write the decrypted value as a new scope-local `<KEY>.age` via `addSecret`. Preserves any notes from the shared entry if available; notes default to `nil` if not.
4. Delete `<KEY>.link`.

### 5. VaultCoreEncryptionTests.swift — encryption test suite

swift-testing `@Suite("VaultCore Encryption")`. Each test uses a fresh `AgeKeyFixture` and a fresh temp vault directory, cleaned up via `defer`.

Tests to cover:

- `addSecret` then `getValue` round-trip — returned value equals original
- `addSecret` with notes — `SecretContent.notes` is preserved after encrypt/decrypt
- `getValue` on a linked key — resolves the `.link`, decrypts the shared entry, returns the shared value
- `rotate` — value changes after rotation, notes are preserved
- `rotateShared` — `getValue` on any scope linking to the shared entry returns the new value after rotation
- `unlink` — after calling `unlink`, a `.age` file exists for the key, the `.link` is gone, and `getValue` (without link resolution) returns the former shared value
- `orphanedSharedEntries` end-to-end — creates shared entries via `addSecret` for shared path directly, verifies orphan detection identifies entries with no `.link` references
- Error: `getValue` on a missing key throws `secretNotFound`
- Error: `getValue` on a linked key where the shared entry is absent throws `linkTargetMissing`
- Error: `rotate` on a nonexistent key throws `secretNotFound`
- Error: `unlink` on a non-linked key throws `secretNotFound`
