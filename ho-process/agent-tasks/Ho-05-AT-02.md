# Ho-05-AT-02 — Read + reveal: secret list, detail pane, Touch-ID reveal, history

## Goal

Fill the center and detail columns of the Workshop. Selecting a scope lists its
secrets (name + value/link indicator, from `inspect`); selecting a secret opens
a detail slide-in showing a masked value that reveals behind Touch ID and stays
revealed until selection changes, plus notes, link target, and rotation history
(git log of the secret's file). Add the one Core capability the history needs:
`Conduit.log`. Read + reveal only — no mutation (that's AT-03).

## Required Changes

### 1. `Conduit.log(fileURL:) -> [CommitInfo]` (Decision 6)

- `CommitInfo` in `ConduitTypes.swift`: `Sendable, Equatable` with `shortSHA:
  String`, `date: String` (ISO, as git prints), `subject: String`.
- `log` shells `git log --follow --format=%h%x1f%ad%x1f%s --date=short --
  <relativePath>` through the existing `git(_:)` helper (vault as working dir),
  splitting on the `%x1f` unit separator per line. Returns `[]` for an
  untracked file (git exits 0 with empty output) — do not throw on "no history".
  Throw `VaultError.gitInvocationFailed` on a non-zero exit that isn't the
  empty-history case, matching sibling ops.
- The `fileURL` is a secret's on-disk path (`<scope>/<KEY>.age` or `.link`);
  compute the vault-relative path for the `-- <path>` argument.

### 2. `WorkshopModel` — listing, reveal, history

- `secrets(for scopeID:) -> [SecretInfo]` via `VaultCore(vaultURL:).inspect`;
  store the selected scope's secrets and a `selectedSecretKey: String?`.
- **Reveal state (Decision 4):** a `revealedValue: String?` plus the key it
  belongs to. `reveal(key:inScope:)` builds the age provider (file-key when
  `devAgeKeyPath` is set, else Keychain), obtains an `AgeKeyHandle`, constructs
  `VaultCore(vaultURL:ageKeyURL: handle.url)`, calls `getValue`, stores the
  plaintext, and **releases the handle** in a `defer`. Changing
  `selectedSecretKey` or `selectedScopeID` clears `revealedValue` (re-mask).
  Touch-ID failure / user-cancel maps to `errorMessage`, leaves value masked.
- `history(for secret:) -> [CommitInfo]` via `Conduit.log(fileURL:)` on the
  secret's file URL; cache per selected secret.
- Keep methods synchronous on `@MainActor` (AT-01 posture). The Keychain prompt
  blocks briefly; acceptable for v1 — do not introduce `async` here.

### 3. Views

- `SecretList`: rows for the selected scope's `inspect` output — key, a glyph
  distinguishing `.value` from `.link(sharedID:)`, bound to `selectedSecretKey`.
  A "select a scope" empty state when none is selected.
- `SecretDetail`: the slide-in. Masked value (`••••••••`) with a Reveal control
  that calls `model.reveal(...)`; when `revealedValue` matches the selected key,
  show the plaintext (with a copy affordance). Show notes (read-only here —
  editing is AT-03), the link target when the secret is `.link(sharedID:)`, and
  the rotation history list from `history(for:)`. No edit or action buttons yet.
- `WorkshopWindow`: put `SecretList` in the content column and `SecretDetail`
  in the detail column.
- All new `View` files coverage-excluded.

### 4. Tests

- `Conduit.log`: in a temp git vault, commit a secret file twice, assert `log`
  returns two `CommitInfo` newest-first with the right subjects; an untracked
  file returns `[]`.
- `WorkshopModel` (file-key path, temp vault with a real age key fixture —
  reuse the Core test fixtures' ephemeral key approach):
  - listing returns the scope's secrets;
  - `reveal` yields the known plaintext for a fixture secret, and the handle is
    released (temp key file gone after the call);
  - changing selection clears `revealedValue`;
  - `history` maps `Conduit.log` output to the detail model.

## Do Not

- Do not modify anything under `Sources/SharibakoCLI/` (Decision 1).
- Do not add any mutation — listing and reveal only. Add/edit/rotate/notes-edit/
  materialize/sync/rescan are AT-03.
- Do not wire the link *picker* — display an existing link target as text only;
  binding a key to a shared entry is ho-07.
- Do not add an auto-hide timer or clipboard-clear on reveal — ho-06 refinement.
  The only re-mask trigger is selection change (Decision 4).
- Do not make the reveal `async` or add a spinner subsystem; the brief Keychain
  block is acceptable for v1.
- Do not print, log, or persist decrypted values anywhere but the in-memory
  `revealedValue`; release the age-key handle on every path.

## Stop Condition

If `git log --follow` on a secret file returns output that doesn't cleanly split
on the chosen separator for real vault history (e.g. a subject containing the
separator byte), stop and surface the parsing choice before shipping a fragile
split — the history is read-only chrome and not worth a brittle parser.
