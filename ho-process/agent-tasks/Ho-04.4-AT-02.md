# Ho-04.4-AT-02 — Rewire `InitCommand`, retire `InteractiveIngestPrompt`

## Goal

Route `sharibako init` through the new `DashboardIngestPrompt` in place of the per-key `InteractiveIngestPrompt`, and remove `InteractiveIngestPrompt` and its tests. Every other part of the `init` flow — first-run key generation, scope-ID/type prompts, reconcile, reporting, the source-`.env` untouched guarantee — stays exactly as it is. Only the decision-collection call changes.

## Context

Read before implementing:

- `ho-process/hos/ho-04.4-ingest-dashboard.md` (parent ho — Decision 5 authorizes the retirement; Decision 7 fixes the seam that stays put)
- `ho-process/hos/ho-04.2-interactive-init.md` (the `init` flow being preserved; Phase 3 Reflect explains why the prompt is replaced, not patched)
- `Sources/SharibakoCLI/Commands/InitCommand.swift` (the command — the only substantive edit is the `_run` default decision source)
- `Sources/SharibakoCLI/Support/DashboardIngestPrompt.swift` and `Sources/SharibakoCLI/Support/IngestDashboard.swift` (AT-01's output — what `init` now drives)
- `Sources/SharibakoCLI/Support/InteractiveIngestPrompt.swift` (the file being deleted)
- `Tests/SharibakoCLITests/InitCommandTests.swift` (end-to-end tests — they inject `ScriptedIngestDecisionSource`, so they should keep passing unchanged; confirm)
- `Tests/SharibakoCLITests/IngestDecisionSourceTests.swift` (contains both `ScriptedIngestDecisionSource` tests, which stay, and `InteractiveIngestPrompt` tests, which are removed)

## Files

- Modify: `Sources/SharibakoCLI/Commands/InitCommand.swift` (swap the default decision source)
- Delete: `Sources/SharibakoCLI/Support/InteractiveIngestPrompt.swift`
- Modify: `Tests/SharibakoCLITests/IngestDecisionSourceTests.swift` (remove the `InteractiveIngestPrompt` test suite; keep the `ScriptedIngestDecisionSource` suite)

## Required Changes

1. **Swap the default decision source in `InitCommand._run`.**
   Change the `decisionSource` parameter default from `InteractiveIngestPrompt()` to `DashboardIngestPrompt()`:

   ```swift
   func _run(
       cwd: URL = URL(fileURLWithPath: FileManager.default.currentDirectoryPath),
       decisionSource: any IngestDecisionSource = DashboardIngestPrompt(),
       lineReader: () -> String? = { readLine() }
   ) throws {
   ```

   Nothing else in `_run` changes. The `sharedIDs` fetch, the `decisionSource.decisions(for:sharedIDs:)` call, and the `acceptIngest` call are all seam-stable (Decision 7). The scope-ID/type prompts, first-run key generation, reconcile filtering, and `reportResult` are untouched.

2. **Delete `InteractiveIngestPrompt.swift`.**
   The per-key walk is superseded (Decision 5). Remove the file. Confirm no other source references `InteractiveIngestPrompt` after the `InitCommand` swap (grep).

3. **Remove the `InteractiveIngestPrompt` tests.**
   In `Tests/SharibakoCLITests/IngestDecisionSourceTests.swift`, delete the suite(s) that exercise `InteractiveIngestPrompt` (the injected-`selectFactory` branch tests). Keep the `ScriptedIngestDecisionSource` suite intact. If removing tests drops any shared private fixture below use, remove the now-orphaned fixture too — but only orphans this change creates.

4. **Verify the end-to-end `init` tests still pass unchanged.**
   `InitCommandTests` drives `_run` with an injected `ScriptedIngestDecisionSource`, so the source swap should not affect them. Do not rewrite them to reference the dashboard — the whole point of the seam is that they don't care which source is live. If any test hard-codes `InteractiveIngestPrompt`, that's a real coupling to fix; otherwise leave them.
